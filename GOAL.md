# 目標

將 otter 打造成以瀏覽器為主要介面的旅行與朋友聚會記帳拆帳 app。

## 產品方向

otter 協助使用者為旅行或聚會建立支出群組、管理成員、記錄共同支出、查看每個人的分帳餘額，並取得清楚的結清建議。

## 技術方向

- 新增 app 程式碼使用 TypeScript。
- 前端以瀏覽器網頁為主要介面。
- 後端 API 與資料保存支援可驗證的本機開發流程。
- 拆帳、金額與驗證邏輯應有測試保護。

## 目前狀態

核心 MVP 已完成：

- 帳號註冊、登入、登出。
- 建立、選擇、重新命名、設定基準貨幣與刪除支出群組。
- 新增、重新命名與刪除未使用成員。
- 記錄、編輯與刪除支出，包含日期、描述、付款人、金額、貨幣與分帳成員。
- 計算餘額並顯示結清建議。
- PostgreSQL persistence、raw SQL migrations、Docker dev flow、CI checks。
- 支出、餘額與結清建議 CSV 匯出，以及列印友善畫面。

## 下一階段優先順序

1. 改善日常使用流程：更少 prompt、更直覺的表單與錯誤提示。
2. 強化資料修正能力：合併成員、封存群組、批次匯入/匯出。
3. 提升正式使用可靠度：自訂或即時匯率、更多資料驗證、備份/還原路徑。
4. 持續補齊支援核心行為的測試、文件與可部署性。

## 進度檢查標準

有意義的變更應至少符合一項：

- 改善支出群組、成員、支出、餘額或結清流程。
- 改善瀏覽器 UI 的可用性、可及性或錯誤處理。
- 改善資料保存、資料修正、安全性或部署可靠度。
- 增加能保護拆帳、金額、驗證或 API 行為的測試。
- 更新文件以降低使用、開發或部署的不確定性。

不視為進展：與產品目標無關的表面調整、未驗證的重構、或增加目前用不到的抽象與依賴。
