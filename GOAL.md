# 目標

將 otter 打造成以網頁為主要介面的旅行與朋友聚會記帳拆帳 app。

## 產品方向

otter 應協助使用者在旅行或朋友聚會中記錄共同支出、查看每個人支付了哪些費用、計算各自的分帳餘額，並清楚知道如何結清款項。

主要使用體驗應透過瀏覽器中的網頁介面完成。

## 技術方向

新增的網頁 app 程式碼應使用 TypeScript；除非有明確理由，主要功能不應以純 JavaScript 實作。

## 近期里程碑

- 建立網頁 app 的基礎架構與開發流程。
- 建立可用的第一個畫面，用於管理旅行或群組。
- 支援新增參與者。
- 支援記錄共同支出，包含付款人、金額、描述與參與者。
- 計算參與者之間的分帳餘額。
- 顯示簡單的結清建議。
- 加入適合本機開發或早期原型的基本資料保存方式。
- 隨著 app 成形，持續更新專案文件。

## 進度檢查標準

檢查 repository 是否往目標前進時，應觀察是否有以下變更：

- 新增或改善網頁應用程式結構。
- 實作旅行、群組、參與者、支出、餘額或結清相關功能。
- 改善瀏覽器介面的可用性。
- 增加支援 app 目標的測試、驗證、資料保存或文件。
- 降低產品範圍或實作路徑的不確定性。

若變更與 app 目標無關、只是沒有改善產品的表面調整，或只是重複改動既有檔案但沒有讓 app 更容易建置、使用或驗證，則不視為有意義的進展。
